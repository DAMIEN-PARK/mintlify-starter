## 7가지 MCP 기본 템플릿

자주 사용하는 도구 조합을 미리 구성한 템플릿을 제공합니다. 템플릿을 기반으로 빠르게 MCP를 만들 수 있습니다.

| 템플릿 | 포함 도구 | 활용 시나리오 |
|--------|-----------|---------------|
| 학습 도우미 | 웹 검색, 위키피디아, 퀴즈 생성 | 개념 탐색 및 자기 평가 |
| 리서치 | 웹 검색, 웹 크롤링, DART 공시 | 정보 수집 및 보고서 작성 |
| 코딩 연습 | 계산기, 파일 읽기, 웹 검색 | 프로그래밍 실습 지원 |
| 글쓰기 | 맞춤법 검사, 문법 교정, 웹 검색 | 글쓰기 품질 향상 |
| 수학 / 과학 | 계산기, 날짜/시간, 차트 시각화 | 수치 계산 및 데이터 시각화 |
| 외국어 학습 | 단어장, 문법 교정, 웹 검색 | 외국어 학습 및 번역 |
| 자유 구성 | 사용자 선택 | 목적에 맞게 자유롭게 조합 |

## MCP 실행 4단계

AI가 MCP 도구를 활용하여 응답을 생성하는 과정입니다.

<Steps>
  <Step title="프롬프트 전송">
    학생이 AI에게 질문 또는 요청을 입력합니다.
  </Step>
  <Step title="도구 선택 판단">
    AI 모델이 요청 내용을 분석하여 어떤 MCP 도구를 사용할지 자동으로 판단합니다.
  </Step>
  <Step title="도구 실행">
    선택된 MCP 도구가 실행되어 외부 데이터(웹 검색 결과, 날씨 정보 등)를 가져옵니다.
  </Step>
  <Step title="응답 생성">
    AI가 도구 실행 결과를 바탕으로 최종 답변을 생성하여 학생에게 전달합니다.
  </Step>
</Steps>

## MCP 만들기 및 강의 공유

<Steps>
  <Step title="목적 설정">
    어떤 학습 활동에 활용할 MCP인지 목적을 먼저 정합니다.

    **예시**: "웹 리서치 + 보고서 자동 작성 실습용"
  </Step>
  <Step title="도구 선택">
    목적에 맞는 MCP 도구를 체크박스로 선택합니다. 기본 템플릿을 먼저 선택한 후 도구를 추가하거나 제거할 수도 있습니다.
  </Step>
  <Step title="이름 & 저장">
    MCP 세트의 이름을 입력하고 **[저장]** 버튼을 클릭합니다.

    <Tip>
      MCP 이름은 수업 주제나 활용 목적이 명확히 드러나도록 작성하세요. 예: `3주차 리서치 실습용 MCP`
    </Tip>
  </Step>
  <Step title="강의 선택 및 공유">
    저장된 MCP 카드에서 **[강의에 공유]** 를 클릭하고 배포할 강의를 선택합니다. 선택 즉시 해당 강의 학생들의 MCP 메뉴에 노출됩니다.
  </Step>
</Steps>

## 에이전트 빌더 활용 4단계

<Steps>
  <Step title="개념 이해">
    워크플로우, 노드, 스킬, 세션 등 에이전트 빌더의 핵심 개념을 먼저 파악합니다.
  </Step>
  <Step title="설계">
    노드를 배치하고 연결하여 AI 처리 흐름을 시각적으로 설계합니다.
  </Step>
  <Step title="테스트">
    설계한 워크플로우를 실행하여 각 노드의 결과를 검증하고 오류를 수정합니다.
  </Step>
  <Step title="배포 & 공유">
    완성된 에이전트를 저장하고 강의에 배포하여 학생들이 사용할 수 있도록 합니다.
  </Step>
</Steps>

## 워크플로우 설계 5단계

<Steps>
  <Step title="노드 추가">
    좌측 노드 패널에서 원하는 노드를 드래그하여 캔버스에 배치합니다. 또는 캔버스 빈 공간을 우클릭하여 노드를 추가할 수도 있습니다.
  </Step>
  <Step title="노드 연결">
    노드의 출력 포트에서 다음 노드의 입력 포트로 선을 드래그하여 연결합니다. 연결선은 데이터 흐름 방향을 나타냅니다.
  </Step>
  <Step title="속성 설정">
    각 노드를 클릭하면 우측에 속성 패널이 열립니다. 노드별 설정(프롬프트 내용, 모델 선택, 조건 식 등)을 입력합니다.
  </Step>
  <Step title="검증">
    **[검증하기]** 버튼을 클릭하면 워크플로우의 연결 오류, 누락된 설정 등을 자동으로 점검합니다.

    | 상태 | 색상 | 의미 |
    |------|------|------|
    | 완료 | 초록 | 노드 정상 실행 완료 |
    | 검사 중 | 노랑 | 현재 노드 실행 진행 중 |
    | 대기 | 회색 | 이전 노드 완료 대기 중 |
    | 오류 | 빨강 | 실행 오류 발생, 확인 필요 |

  </Step>
  <Step title="저장 / 게시">
    검증 완료 후 **[저장]** 으로 임시 저장하거나 **[게시]** 로 공개 상태로 전환합니다. 게시된 에이전트만 학생들에게 배포할 수 있습니다.
  </Step>
</Steps>

## 스킬 만들기 (AI 도움받기)

AI의 도움을 받아 스킬을 빠르게 생성할 수 있습니다.

<Steps>
  <Step title="자연어 설명">
    스킬 생성 창에서 만들고 싶은 스킬을 자연어로 설명합니다.

    **예시**: "사용자의 텍스트를 입력받아 핵심 키워드 5개를 추출하고 중요도 순으로 나열하는 스킬"
  </Step>
  <Step title="자동 생성">
    AI가 설명을 분석하여 스킬 구조와 프롬프트를 자동으로 생성합니다.
  </Step>
  <Step title="예시 추가 / 개선">
    생성된 스킬에 Few-shot 예시를 추가하거나 프롬프트를 직접 수정하여 성능을 개선합니다.
  </Step>
  <Step title="저장">
    완성된 스킬을 저장합니다. 저장된 스킬은 스킬 라이브러리에 보관되어 이후 워크플로우에서 재사용할 수 있습니다.
  </Step>
</Steps>

    스킬 라이브러리에서 이름 또는 태그로 스킬을 검색할 수 있습니다. 최신순, 이름순, 사용 횟수순으로 정렬하여 원하는 스킬을 빠르게 찾으세요.
  </Tab>
  <Tab title="바로 적용">
    워크플로우 편집 중 스킬 노드를 추가할 때, 라이브러리에서 저장된 스킬을 선택하면 즉시 노드에 적용됩니다.
  </Tab>
  <Tab title="수정 / 관리">
    라이브러리의 스킬을 클릭하면 이름, 설명, 프롬프트를 수정할 수 있습니다. 수정 사항은 해당 스킬을 사용하는 모든 워크플로우에 반영됩니다.
  </Tab>
  <Tab title="복제">
    기존 스킬을 복제하여 변형 버전을 만들 수 있습니다. 유사한 목적의 스킬 변형 작업 시 유용합니다.
  </Tab>
</Tabs>

## 테스트 & 실행

<AccordionGroup>
  <Accordion title="검증하기" icon="circle-check">
    **[검증하기]** 버튼으로 워크플로우의 구성 오류를 사전에 점검합니다. 연결이 끊긴 노드, 필수 설정 누락 등을 자동으로 감지합니다.
  </Accordion>
  <Accordion title="노드별 결과 확인" icon="eye">
    테스트 실행 중 각 노드가 처리한 입력값과 출력값을 노드별로 확인할 수 있습니다. 데이터가 워크플로우를 따라 어떻게 변환되는지 추적하세요.
  </Accordion>
  <Accordion title="오류 디버깅" icon="bug">
    오류(빨강) 상태의 노드를 클릭하면 오류 메시지와 원인이 표시됩니다. 속성 패널에서 설정을 수정하고 다시 실행하여 오류를 해결하세요.

    <Warning>
      테스트 실행 중 오류가 발생해도 이전 실행 기록은 보존됩니다. 수정 후 재실행으로 결과를 비교할 수 있습니다.
    </Warning>
  </Accordion>
  <Accordion title="게시" icon="rocket">
    테스트가 완료되고 정상 작동을 확인한 후 **[게시]** 버튼을 클릭합니다. 게시 완료된 에이전트만 학생들에게 배포할 수 있습니다.

    <Note>
      게시 상태로 전환하면 학생들이 에이전트를 즉시 사용할 수 있습니다. 수업 전 반드시 테스트를 완료하고 게시하세요.
    </Note>
  </Accordion>
</AccordionGroup>

## 내 에이전트 관리

에이전트 빌더 메뉴의 **[내 에이전트]** 탭에서 생성한 모든 에이전트를 관리합니다.

<Tabs>
  <Tab title="목록">
    생성한 에이전트 전체 목록을 카드 또는 테이블 형식으로 확인합니다. 이름, 마지막 수정일, 게시 상태가 표시됩니다.
  </Tab>
  <Tab title="수업 공유 탭">
    현재 학생들에게 배포 중인 에이전트 목록을 별도 탭으로 확인합니다. 강의별로 배포된 에이전트를 한눈에 파악할 수 있습니다.
  </Tab>
  <Tab title="검색 / 필터">
    에이전트 이름으로 검색하거나 게시 상태(게시됨/임시저장), 강의별 필터를 적용하여 원하는 에이전트를 빠르게 찾습니다.
  </Tab>
  <Tab title="관리 기능">
    각 에이전트 카드에서 다음 작업을 수행할 수 있습니다.

    | 기능 | 설명 |
    |------|------|
    | 대화 | 에이전트와 직접 대화 (테스트 목적) |
    | 노드 편집 | 워크플로우 편집 화면으로 이동 |
    | 공유 | 강의 선택 후 학생들에게 배포 |
    | 비활성화 | 게시를 중단하여 학생 사용 불가 처리 |
    | 삭제 | 에이전트 영구 삭제 (되돌릴 수 없음) |
  </Tab>
</Tabs>

<Warning>
  에이전트 삭제는 되돌릴 수 없습니다. 삭제 전 노드 편집 화면에서 워크플로우 구조를 스크린샷으로 저장하거나, 복제 기능을 사용하여 백업하는 것을 권장합니다.
</Warning>
