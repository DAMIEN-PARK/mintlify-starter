## 학습 자료 관리 개요

강사는 수업에 활용할 **프롬프트 템플릿**과 **Few-shot 예시 세트**를 직접 작성하고 강의별로 배포할 수 있습니다. 배포된 자료는 학생의 프롬프트 메뉴에 즉시 노출됩니다.

<CardGroup cols={2}>
  <Card title="프롬프트 템플릿" icon="pen-nib">
    자주 사용하는 프롬프트 구조를 템플릿으로 저장하고 공유합니다. 변수 삽입을 통해 재사용 가능한 범용 템플릿을 만들 수 있습니다.
  </Card>
  <Card title="Few-shot 예시 세트" icon="list-check">
    입력/출력 예시 쌍을 묶어 하나의 세트로 관리합니다. AI 모델의 답변 방향성을 잡아주는 학습 가이드 역할을 합니다.
  </Card>
</CardGroup>

---

## 탭 전환

학습 자료 관리 페이지에는 두 개의 탭이 있습니다.

<Tabs>
  <Tab title="프롬프트 템플릿">
    작성된 프롬프트 템플릿 목록을 확인하고 새 템플릿을 추가합니다. 각 템플릿 카드에서 공유 강의를 설정할 수 있습니다.
  </Tab>
  <Tab title="Few-shot 예시">
    작성된 Few-shot 예시 세트 목록을 확인하고 새 세트를 추가합니다. 각 세트에 여러 입력/출력 예시 쌍을 등록하여 관리합니다.
  </Tab>
</Tabs>

## 프롬프트 템플릿 공유

### 템플릿 작성

<Steps>
  <Step title="새 템플릿 만들기">
    프롬프트 템플릿 탭에서 **[+ 새로 만들기]** 버튼을 클릭합니다. 템플릿 작성 폼이 열립니다.
  </Step>
  <Step title="기본 정보 입력">
    아래 항목을 입력합니다.

    | 항목 | 설명 |
    |------|------|
    | 이름 | 템플릿 카드에 표시될 이름 |
    | 설명 | 이 템플릿의 사용 목적 또는 사용 방법 안내 |
    | 본문 | 실제 프롬프트 내용 (변수 포함 가능) |

  </Step>
  <Step title="변수 삽입 (선택)">
    템플릿 본문에 `[변수명]` 형식으로 변수를 삽입하면 학생이 사용 시 해당 부분을 직접 입력할 수 있는 동적 템플릿이 됩니다.

    **예시**:
    ```
    당신은 노련한 프로젝트 매니저입니다.
    [프로젝트_개요]를 바탕으로 다음 내용을 작성해주세요:
    1. 프로젝트 목표 및 범위
    2. 주요 마일스톤 (총 [기간] 기준)
    3. 리스크 요소 및 대응 방안
    ```

    <Tip>
      변수명은 이해하기 쉽게 명사형으로 작성하세요. 예: `[프로젝트_개요]`, `[대상_독자]`, `[기간]`
    </Tip>
  </Step>
  <Step title="공유 강의 선택 및 저장">
    하단의 강의 목록에서 체크박스로 공유할 강의를 선택하고 **[저장]** 을 클릭합니다. 선택한 강의의 학생 화면에 즉시 노출됩니다.
  </Step>
</Steps>

### 프롬프트 템플릿 예시

<AccordionGroup>
  <Accordion title="시스템 프롬프트 - 프로젝트 매니저" icon="briefcase">
    ```
    당신은 노련한 프로젝트 매니저입니다.
    [프로젝트_개요]를 작성해주세요.

    작성 시 다음 사항을 포함하세요:
    - 프로젝트 목표 및 핵심 성과 지표(KPI)
    - 주요 이해관계자 및 역할
    - 단계별 일정 계획
    - 예상 리스크 및 완화 방안
    ```
  </Accordion>
  <Accordion title="시스템 프롬프트 - 글쓰기 코치" icon="pen">
    ```
    당신은 전문 글쓰기 코치입니다.
    아래 [초안]을 검토하고 다음 관점에서 피드백을 제공하세요:

    1. 논리적 흐름과 구조
    2. 문장의 명확성과 간결함
    3. 독자를 설득하는 표현 방식
    4. 개선이 필요한 구체적 부분 (인용 후 수정 예시 제시)
    ```
  </Accordion>
  <Accordion title="시스템 프롬프트 - 코드 리뷰어" icon="code">
    ```
    당신은 시니어 소프트웨어 엔지니어입니다.
    아래 [코드]를 리뷰하고 다음 항목을 평가하세요:

    - 코드 품질 및 가독성
    - 잠재적 버그 또는 엣지 케이스
    - 성능 최적화 가능성
    - 보안 취약점 (해당 시)
    - 개선된 코드 예시 제안
    ```
  </Accordion>
</AccordionGroup>

### 템플릿 카드 관리

작성된 템플릿은 카드 형태로 표시됩니다. 각 카드에서 다음 작업을 수행할 수 있습니다.

<CardGroup cols={3}>
  <Card title="복제" icon="copy">
    기존 템플릿을 복사하여 새 템플릿의 시작점으로 활용합니다. 유사한 템플릿을 여러 개 만들 때 유용합니다.
  </Card>
  <Card title="수정" icon="pen-to-square">
    템플릿 이름, 설명, 본문 내용을 수정합니다. 수정 후 저장하면 공유된 강의의 학생 화면에 즉시 반영됩니다.
  </Card>
  <Card title="삭제" icon="trash">
    템플릿을 삭제합니다. 삭제된 템플릿은 학생 화면에서도 사라집니다.

    <Warning>
      삭제는 되돌릴 수 없습니다. 필요한 경우 먼저 복제한 후 삭제하세요.
    </Warning>
  </Card>
</CardGroup>

## Few-shot 예시 공유

### Few-shot 예시란?

Few-shot 학습은 AI 모델에게 몇 가지 예시 입출력을 제공하여 원하는 답변 방향성을 유도하는 기법입니다.

<CardGroup cols={2}>
  <Card title="활용 목적" icon="bullseye">
    - AI 답변의 형식과 스타일을 일관되게 유지
    - 특정 도메인이나 수업 맥락에 맞는 응답 유도
    - 학생들이 프롬프트 설계 원리를 직접 체험
  </Card>
  <Card title="기대 효과" icon="chart-line">
    - 모델 성능 향상 및 답변 품질 개선
    - 답변 스타일 일관성 확보
    - 학생들의 Few-shot 기법 이해 촉진
  </Card>
</CardGroup>

---

### Few-shot 예시 세트 작성

<Steps>
  <Step title="새 세트 만들기">
    Few-shot 예시 탭에서 **[+ 새로 만들기]** 버튼을 클릭합니다. 세트 작성 폼이 열립니다.
  </Step>
  <Step title="세트 이름 입력">
    세트의 목적이나 주제를 나타내는 이름을 입력합니다.

    **예시**: `감성 분석 예시`, `이메일 작성 스타일`, `수학 풀이 단계별 설명`
  </Step>
  <Step title="입력/출력 예시 쌍 등록">
    **[+ 예시 추가]** 버튼을 클릭하여 입력과 출력 쌍을 등록합니다. 세트 하나에 여러 쌍을 추가할수록 모델 유도 효과가 높아집니다.

    | 항목 | 설명 | 예시 |
    |------|------|------|
    | 입력 (Input) | 사용자가 AI에게 보내는 질문이나 요청 | "이 제품 정말 별로예요" |
    | 출력 (Output) | AI가 반환해야 할 이상적인 답변 | "감성: 부정 / 강도: 강함" |

    <Tip>
      예시는 최소 3~5쌍 이상 등록하는 것이 효과적입니다. 다양한 케이스를 포함하면 모델이 패턴을 더 잘 파악합니다.
    </Tip>
  </Step>
  <Step title="공유 강의 선택 및 배포">
    하단에서 배포할 강의를 체크박스로 선택하고 **[저장]** 을 클릭합니다. 선택한 강의 학생들에게 즉시 배포됩니다.
  </Step>
</Steps>

### Few-shot 세트 활용 예시

<AccordionGroup>
  <Accordion title="감성 분석" icon="face-smile">
    **입력/출력 쌍 예시**:

    | 입력 | 출력 |
    |------|------|
    | 이 제품 정말 별로예요 | 감성: 부정 / 강도: 강함 |
    | 생각보다 훨씬 좋네요! | 감성: 긍정 / 강도: 강함 |
    | 그냥 평범한 것 같아요 | 감성: 중립 / 강도: 약함 |
  </Accordion>
  <Accordion title="번역 스타일 통일" icon="language">
    **입력/출력 쌍 예시**:

    | 입력 | 출력 |
    |------|------|
    | Hello, how are you? | 안녕하세요, 어떻게 지내세요? |
    | Good morning! | 좋은 아침이에요! |
    | Thank you so much. | 정말 감사합니다. |
  </Accordion>
  <Accordion title="수학 풀이 형식 통일" icon="square-root-variable">
    **입력/출력 쌍 예시**:

    | 입력 | 출력 |
    |------|------|
    | 2 + 3 × 4 = ? | [풀이] 곱셈 우선: 3 × 4 = 12 → 2 + 12 = 14 [정답] 14 |
    | (5 + 3) × 2 = ? | [풀이] 괄호 우선: 5 + 3 = 8 → 8 × 2 = 16 [정답] 16 |
  </Accordion>
</AccordionGroup>
