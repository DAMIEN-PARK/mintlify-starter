## 강의 개설 프로세스

GrowFit에서 강의를 개설하면 독립된 학습 공간이 만들어지고 학생 초대 코드가 자동으로 발급됩니다.

<Steps>
  <Step title="메뉴 진입">
    좌측 사이드바에서 **[강의 등록]** 을 클릭합니다. 개설된 강의 목록 페이지로 이동합니다.
  </Step>
  <Step title="새 강의 등록">
    강의 목록 페이지 우측 상단의 **[새 강의 등록]** 버튼을 클릭합니다. 강의 등록 양식이 열립니다.
  </Step>
  <Step title="기본 정보 설정">
    강의에 필요한 정보를 입력합니다.

    | 항목 | 설명 |
    |------|------|
    | 강의명 | 학생 화면에 표시될 강의 이름 |
    | 강의 기간 | 시작일 ~ 종료일 설정 |
    | 최대 학생 수 | 강의 참여 가능한 최대 인원 |
    | AI 모델 | 학생이 사용할 기본 AI 모델 선택 |
    | 소속 | 소속 기관명 입력 |

    <Note>
      소속이 없는 경우 소속 항목에 **'무소속 과정'** 을 입력하세요.
    </Note>
  </Step>
  <Step title="강의 생성 완료">
    모든 정보 입력 후 **[강의 생성]** 버튼을 클릭합니다. 강의가 생성되면 **초대 코드가 자동으로 발급**됩니다. 발급된 초대 코드를 학생들에게 공유하세요.

    <Tip>
      초대 코드와 함께 초대 링크도 발급됩니다. 링크를 직접 공유하면 학생들이 코드를 별도로 입력할 필요 없이 바로 강의에 참여할 수 있습니다.
    </Tip>
  </Step>
</Steps>

## 강의 기본 정보 상세

<AccordionGroup>
  <Accordion title="강의명 설정" icon="pen">
    강의명은 학생 화면의 강의 목록, 통계 페이지 등 곳곳에 표시됩니다. 학생들이 이해하기 쉽고 구분이 명확한 이름을 사용하세요.

    **예시**: `2024년 1학기 AI 활용 실습`, `프롬프트 엔지니어링 기초반`
  </Accordion>
  <Accordion title="강의 기간 설정" icon="calendar">
    강의 기간은 시작일과 종료일로 설정합니다. 종료일이 지나면 강의 상태가 자동으로 **'종료'** 로 전환되며, 학생들의 새로운 활동이 제한됩니다.

    - 기간 연장은 강의 설정에서 언제든 수정 가능합니다.
    - 종료 후에도 학습 기록은 열람 가능합니다.
  </Accordion>
  <Accordion title="AI 모델 선택" icon="robot">
    강의에서 기본으로 사용할 AI 모델을 선택합니다. 학생들은 이 모델을 기본값으로 사용하게 됩니다.

    - 강의 개설 후에도 모델 변경이 가능합니다.
    - 여러 모델을 허용하는 경우 학생이 직접 선택할 수 있습니다.
  </Accordion>
  <Accordion title="최대 학생 수 설정" icon="users">
    강의에 참여 가능한 최대 학생 수를 설정합니다. 설정한 인원에 도달하면 추가 초대 코드 입력이 제한됩니다.

    - 인원 초과 시 강의 설정에서 최대 인원을 늘릴 수 있습니다.
  </Accordion>
</AccordionGroup>

## 초대 코드 관리

<Tabs>
  <Tab title="초대 코드 확인">
    강의 생성 직후 자동으로 초대 코드가 발급됩니다. 이후에도 **강의 목록 > 해당 강의 클릭 > [초대 코드 확인]** 에서 언제든 확인할 수 있습니다.

    - 초대 코드는 영문+숫자 조합의 고유한 문자열입니다.
    - 같은 강의의 초대 코드는 동일하게 유지됩니다.
  </Tab>
  <Tab title="초대 링크 공유">
    초대 링크를 학생에게 전달하면 링크 클릭만으로 강의에 바로 참여할 수 있습니다.

    - **[링크 복사]** 버튼으로 클립보드에 복사
    - 이메일, 메신저, LMS 등 어떤 채널로든 공유 가능
    - 링크에는 초대 코드가 포함되어 있어 자동 입력됩니다.
  </Tab>
  <Tab title="코드 재발급">
    보안상의 이유로 기존 초대 코드를 무효화하고 새 코드를 발급해야 할 경우, **강의 설정 > [초대 코드 재발급]** 을 사용하세요.

    <Warning>
      초대 코드를 재발급하면 기존 코드는 즉시 사용 불가가 됩니다. 이미 참여한 학생들에게는 영향이 없지만, 아직 참여하지 않은 학생들에게 새 코드를 공유해야 합니다.
    </Warning>
  </Tab>
</Tabs>

## 강의 관리 기능

<CardGroup cols={2}>
  <Card title="강의 수정" icon="pen-to-square">
    **강의 설정 > 정보 수정** 에서 강의명, 기간, 학생 수, 모델 등 기본 정보를 수정하고 저장합니다. 수정 사항은 즉시 반영됩니다.
  </Card>
  <Card title="강의 삭제" icon="trash">
    강의를 삭제하면 강의와 연관된 모든 데이터가 **영구적으로 삭제**됩니다. 학생 기록, 사용 통계, 배포된 자료가 모두 사라지므로 신중하게 진행하세요.

    <Warning>
      강의 삭제는 되돌릴 수 없습니다. 삭제 전 필요한 데이터를 반드시 내보내거나 백업하세요.
    </Warning>
  </Card>
</CardGroup>

## 강의 상태 관리

강의는 운영 상황에 따라 두 가지 상태로 관리됩니다.

<Tabs>
  <Tab title="진행 중">
    **강의가 활성화된 상태**입니다.

    - 훈련생(수강생)의 참여 가능
    - 모든 학습 기능 활성화 (LLM 실습, 프롬프트, RAG, MCP, 에이전트 등)
    - 강사가 배포한 자료 및 에이전트 사용 가능
    - 학생 초대 코드 유효
    - 실시간 사용량 통계 집계

    <Tip>
      강의 기간 내에 있으면 자동으로 '진행 중' 상태가 유지됩니다.
    </Tip>
  </Tab>
  <Tab title="종료">
    **강의가 완료된 상태**입니다.

    - 학습 기록 및 통계 열람은 계속 가능
    - 새로운 AI 대화 및 학습 활동 불가
    - 초대 코드 비활성화 (새로운 학생 참여 불가)
    - 강사는 학습 결과 리포트를 계속 조회 가능

    <Note>
      강의 종료 후에도 학습 데이터는 보존됩니다. 필요 시 강의 기간을 연장하여 다시 '진행 중' 상태로 전환할 수 있습니다.
    </Note>
  </Tab>
</Tabs>

## 강의 목록 화면 구성

강의 목록 페이지에서 개설한 모든 강의를 한눈에 확인할 수 있습니다.

| 컬럼 | 설명 |
|------|------|
| 강의명 | 개설한 강의의 이름 |
| 기간 | 강의 시작일 ~ 종료일 |
| 학생 수 | 현재 참여 중인 학생 수 / 최대 인원 |
| 상태 | 진행 중 / 종료 배지 |
| 초대 코드 | 클릭 시 코드 및 링크 확인 |
| 관리 | 수정, 삭제 등 관리 버튼 |
