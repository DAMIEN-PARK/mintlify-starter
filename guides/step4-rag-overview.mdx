## 지식베이스(RAG)란?

**지식베이스**는 나만의 문서 저장소입니다. PDF, DOCX 등 파일을 업로드하면 AI가 해당 문서를 검색하여 답변에 활용합니다.

**RAG = 검색 증강 생성 (Retrieval-Augmented Generation)**

> AI가 직접 학습하지 않은 내용도, 내가 업로드한 문서를 기반으로 정확하게 답변할 수 있습니다.

<CardGroup cols={2}>
  <Card title="RAG 없이" icon="xmark" color="#ef4444">
    AI의 학습 데이터에만 의존합니다. 내부 문서, 최신 정보, 회사 규정 등 **AI가 학습하지 않은 내용은 모릅니다.**
  </Card>
  <Card title="RAG 사용 시" icon="check" color="#22c55e">
    업로드한 문서를 실시간으로 검색하여 답변합니다. **내 문서 기반의 정확한 답변**을 얻을 수 있습니다.
  </Card>
</CardGroup>

---

## 왜 지식베이스가 필요한가요?

<AccordionGroup>
  <Accordion title="AI는 내부 자료를 모릅니다">
    일반 AI 모델은 공개된 데이터로 학습되었기 때문에, 여러분 회사의 내부 규정, 사내 문서, 팀 전용 자료는 전혀 알 수 없습니다. 지식베이스에 업로드하면 AI가 해당 내용을 참조할 수 있게 됩니다.
  </Accordion>
  <Accordion title="AI의 지식에는 학습 시점 제한이 있습니다">
    모든 AI 모델에는 **학습 데이터 마감일(Knowledge Cutoff)**이 있습니다. 그 이후 발표된 최신 자료, 개정된 법규, 업데이트된 제품 사양 등은 AI가 알지 못합니다. 최신 문서를 지식베이스에 등록하면 이 문제를 해결할 수 있습니다.
  </Accordion>
  <Accordion title="AI 환각(Hallucination)을 줄입니다">
    모르는 것을 아는 척 답변하는 AI의 "환각" 현상은 신뢰성을 떨어뜨립니다. 지식베이스를 제공하면 AI가 **실제 문서를 근거로** 답변하므로 환각 발생률이 크게 줄어듭니다.
  </Accordion>
</AccordionGroup>

## 지식베이스 설정 방식 비교

<Tabs>
  <Tab title="기본 설정 (권장)">
    **처음 사용자에게 권장하는 방법입니다.**

    <Steps>
      <Step title="파일 업로드">
        PDF, DOCX, TXT 등 문서를 지식베이스에 업로드합니다.
      </Step>
      <Step title="기본 설정으로 생성">
        청킹, 검색, 임베딩 설정을 기본값으로 유지하고 생성합니다.
      </Step>
      <Step title="AI 실습에서 참조">
        AI 실습 화면에서 해당 지식베이스를 선택하여 대화에 활용합니다.
      </Step>
    </Steps>

    <Tip>
      대부분의 경우 기본 설정만으로도 충분한 성능을 발휘합니다. 결과가 만족스럽지 않을 때 고급 옵션을 조정해보세요.
    </Tip>
  </Tab>
  <Tab title="고급 설정 (옵션)">
    **성능을 더 세밀하게 조정하고 싶을 때 사용합니다.**

    | 설정 항목 | 설명 |
    |---|---|
    | 청킹(Chunking) 설정 | 문서를 나누는 크기와 방식 조정 |
    | 검색 설정 | Top-K, 유사도 임계값 조정 |
    | 리랭크 모델 활성화 | 검색 결과의 순위를 재정렬하여 정확도 향상 |

    <Note>
      고급 설정은 문서 유형, 질문 패턴, 원하는 정확도에 따라 최적값이 달라집니다. 비교 모드를 활용하여 설정 변경 전후 성능을 비교해보세요.
    </Note>
  </Tab>
</Tabs>

## 청크(Chunk) 개념 이해하기

AI는 업로드된 문서 전체를 한 번에 읽는 것이 아니라, **질문과 관련된 부분만 찾아서** 답변에 활용합니다. 이를 위해 문서를 작은 조각(청크)으로 나누는 과정이 필요합니다.

<CardGroup cols={2}>
  <Card title="청크 없이 (비효율)" icon="book">
    **책 전체를 던져주는 것**과 같습니다. AI가 방대한 내용 중 어디를 참조해야 할지 혼란스러워집니다.
  </Card>
  <Card title="청킹 후 (효율적)" icon="file">
    **필요한 페이지만 펼쳐주는 것**과 같습니다. AI가 질문과 관련된 정확한 부분만 참조하여 답변합니다.
  </Card>
</CardGroup>

**청킹(Chunking)**: 문서를 검색하기 좋은 크기로 나누는 과정

<AccordionGroup>
  <Accordion title="청크 크기(Chunk Size)">
    문서를 나누는 기준 길이입니다. 너무 작으면 문맥이 단절되고, 너무 크면 검색 정확도가 떨어집니다.

    | 청크 크기 | 특징 | 적합한 문서 |
    |---|---|---|
    | 소형 (256~512 토큰) | 세밀한 검색, 빠른 처리 | FAQ, 짧은 규정 문서 |
    | 중형 (512~1024 토큰) | 균형 잡힌 성능 **(기본값 권장)** | 일반 보고서, 매뉴얼 |
    | 대형 (1024~2048 토큰) | 풍부한 문맥 | 논문, 긴 계약서 |
  </Accordion>
  <Accordion title="청크 중첩(Overlap)">
    인접한 청크 사이에 **겹치는 구간**을 두어 문맥 단절을 방지합니다.

    예: 청크 크기 500 토큰, 중첩 50 토큰
    - 청크 1: 1~500 토큰
    - 청크 2: 450~950 토큰 (앞 청크 끝 50 토큰 포함)
    - 청크 3: 900~1400 토큰

    중첩 덕분에 청크 경계에서 잘린 문장도 다음 청크에서 이어서 읽을 수 있습니다.
  </Accordion>
</AccordionGroup>

## 지식베이스 생성 4단계

<Steps>
  <Step title="데이터 소스 설정">
    문서 파일을 업로드합니다. 지원 형식: PDF, DOCX, TXT, MD 등

    업로드 시 자동으로 텍스트를 추출합니다.
  </Step>
  <Step title="청킹 설정">
    문서를 어떤 크기로 나눌지, 중첩 구간을 얼마나 둘지 설정합니다.

    **처음에는 기본값 유지를 권장합니다.**
  </Step>
  <Step title="검색 설정">
    AI가 몇 개의 청크를 참조할지(Top-K), 유사도 기준은 어느 정도로 할지(Threshold) 설정합니다.
  </Step>
  <Step title="미리보기 & 확인">
    설정된 내용을 미리보기로 확인하고 지식베이스를 생성합니다.

    생성 완료 후 AI 실습 화면에서 바로 활용할 수 있습니다.
  </Step>
</Steps>

## 핵심 기술 용어

<AccordionGroup>
  <Accordion title="임베딩 (Embedding)">
    텍스트를 **숫자(벡터)로 변환**하는 기술입니다. AI는 텍스트를 직접 비교할 수 없으므로, 의미를 담은 숫자 배열로 변환하여 유사한 의미를 가진 텍스트를 찾습니다.

    - **기본값**: `text-embedding-3-small`
    - 의미가 비슷한 문장은 벡터 공간에서 가까운 위치에 배치됩니다.
    - 예: "강아지"와 "개"는 벡터 공간에서 서로 가까운 위치

    <Note>
      임베딩 모델을 바꾸면 검색 품질이 달라질 수 있습니다. 특수한 도메인(의료, 법률 등) 문서에는 해당 분야에 특화된 임베딩 모델이 더 효과적일 수 있습니다.
    </Note>
  </Accordion>
  <Accordion title="리랭크 (Rerank)">
    1차 검색으로 찾아낸 청크들의 **순위를 다시 정렬**하는 과정입니다. 단순 유사도 검색만으로는 놓칠 수 있는 관련성 높은 청크를 상위로 올려줍니다.

    - **기본값**: `BAAI/bge-reranker-v2-m3`
    - 활성화하면 검색 정확도가 높아지지만, 처리 시간이 소폭 증가합니다.
    - 중요도가 높은 문서 검색이나 정밀한 답변이 필요할 때 활성화 권장
  </Accordion>
  <Accordion title="Top-K (참조 청크 수)">
    AI가 답변 생성 시 참조하는 **청크의 최대 개수**입니다.

    | Top-K 값 | 특징 | 권장 상황 |
    |---|---|---|
    | 3~5개 | 정확한 답변, 빠른 처리 | 구체적 사실 확인 |
    | 5~7개 | 균형 잡힌 성능 **(권장)** | 일반적인 질의응답 |
    | 10개 이상 | 폭넓은 참조, 노이즈 증가 | 종합 분석, 리포트 작성 |

    Top-K를 너무 높이면 관련 없는 청크도 포함되어 답변 품질이 오히려 떨어질 수 있습니다.
  </Accordion>
  <Accordion title="유사도 임계값 (Similarity Threshold)">
    검색 결과를 포함할 **최소 유사도 기준**입니다. 0~1 사이의 값으로 설정합니다.

    - **높게 설정 (0.8 이상)**: 매우 관련성 높은 청크만 포함 → 정확하지만 정보 누락 가능
    - **낮게 설정 (0.5 이하)**: 더 많은 청크 포함 → 폭넓지만 노이즈 증가

    **권장값**: 0.6~0.7 (정확도와 포괄성의 균형)
  </Accordion>
</AccordionGroup>

## 비교 모드 활용하기

비교 모드에서는 **2개 패널을 나란히 배치**하여 다른 설정의 AI 응답을 실시간으로 비교할 수 있습니다.

<CardGroup cols={3}>
  <Card title="순수 LLM" icon="brain">
    외부 문서 참조 없이 AI의 **기본 학습 지식만** 사용합니다. RAG 효과를 확인하기 위한 기준선으로 활용합니다.
  </Card>
  <Card title="문서 참조" icon="file-lines">
    **기본 RAG 설정**으로 업로드한 파일을 불러옵니다. 지식베이스 활용 시 기본 성능을 확인합니다.
  </Card>
  <Card title="RAG 설정" icon="sliders">
    Top-K, 임계값, 리랭크 등 **세부 파라미터를 직접 조절**합니다. 설정 변경이 답변에 미치는 영향을 비교합니다.
  </Card>
</CardGroup>

## 지식베이스 활용 전후 비교 예시

**질문**: "우리 회사의 연차 휴가 사용 규정을 알려줘."

<Tabs>
  <Tab title="RAG 없이 (순수 LLM)">
    **AI 답변:**

    ```
    일반적으로 근로기준법에 따르면, 1년 이상 근무한 직원은
    연간 15일의 유급 연차휴가를 받을 수 있습니다.
    3년 이상 근무 시 2년마다 1일씩 추가되어 최대 25일까지
    받을 수 있습니다...
    ```

    **문제점**: 일반적인 법령 정보만 제공. 회사별 세부 규정(사용 기간, 잔여 처리, 사전 승인 절차 등)을 모름.
  </Tab>
  <Tab title="RAG 사용 후 (문서 참조)">
    **AI 답변:**

    ```
    [인사 규정 5장 연차 휴가 조항 참조]

    당사의 연차 휴가 규정은 다음과 같습니다:
    - 사용 신청: 최소 3일 전 근태 시스템에 등록
    - 당해 연도 미사용 연차: 다음 연도 3월까지 이월 가능 (최대 5일)
    - 연차 5일 이상 연속 사용 시 팀장 사전 승인 필요
    - 미사용 연차 수당 지급 기준: 연도 말 잔여분의 50%...
    ```

    **개선점**: 실제 사내 규정을 근거로 정확한 회사별 세부 사항 제공. 문서 출처 명시.
  </Tab>
</Tabs>

---

<Note>
  지식베이스 RAG는 단순한 문서 검색을 넘어, AI가 **"내 맥락"을 이해하고 답변**하게 만드는 핵심 기술입니다. 사내 규정, 교육 자료, 업무 매뉴얼을 등록하면 AI가 여러분의 업무 환경에 맞춤화된 도우미로 변화합니다.
</Note>
