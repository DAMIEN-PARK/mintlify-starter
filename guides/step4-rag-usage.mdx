## 지식베이스 RAG란?

**RAG(Retrieval-Augmented Generation)**는 AI가 학습 데이터 외에 직접 업로드한 문서를 검색하여 더 정확한 답변을 생성하는 기술입니다. 사내 자료, 강의 교안, 연구 논문 등 나만의 문서를 AI에게 연결하면 훨씬 구체적이고 신뢰도 높은 답변을 얻을 수 있습니다.

---

## 간편 업로드

가장 빠르게 문서를 업로드하고 AI에 연결하는 방법입니다.

<Steps>
  <Step title="지식베이스 메뉴 클릭">
    사이드바에서 **지식베이스** 메뉴를 클릭합니다.
  </Step>
  <Step title="간편 업로드 버튼 선택">
    화면 상단 또는 중앙의 **간편 업로드** 버튼을 선택합니다.
  </Step>
  <Step title="파일 업로드 → 자동 임베딩">
    파일을 **드래그 앤 드롭**하거나 클릭하여 업로드합니다. 업로드가 완료되면 자동으로 임베딩(벡터화) 처리가 시작됩니다.

    <Note>
      임베딩이란 문서 내용을 AI가 검색할 수 있는 수치 형태로 변환하는 과정입니다. 파일 크기에 따라 수 초~수 분이 소요될 수 있습니다.
    </Note>
  </Step>
  <Step title="AI 실습에서 지식베이스 연결">
    AI 실습 화면에서 입력창 왼쪽의 **[+] 버튼**을 클릭합니다.
    - **지식베이스 연결** 선택
    - 업로드한 파일에 **체크**
    - 질문하면 AI가 해당 문서를 참조하여 답변합니다.
  </Step>
</Steps>

---

## 고급 설정

청킹 방식, 인덱스 모드, 리랭크 등 세밀한 설정으로 RAG 품질을 높이는 방법입니다.

<Steps>
  <Step title="고급 설정 탭 진입">
    **지식베이스 메뉴** → **고급 설정** 탭을 클릭합니다.
  </Step>
  <Step title="파일 선택">
    설정을 적용할 파일을 선택한 후 **다음 단계**로 진행합니다.
  </Step>
  <Step title="청킹 방식 설정">
    문서를 어떻게 조각낼지 설정합니다.

    | 항목 | 설명 | 권장값 |
    |---|---|---|
    | 청킹 방식 | 일반 / 부모-자식 | 부모-자식 (긴 문서에 유리) |
    | 최대 길이 | 청크 하나의 최대 글자 수 | 800 |
    | 중첩 | 청크 간 겹치는 글자 수 | 200 |

    <Tip>
      중첩(Overlap)을 설정하면 청크 경계에서 문맥이 잘리는 문제를 줄일 수 있습니다. 200 전후가 일반적으로 적합합니다.
    </Tip>
  </Step>
  <Step title="프리뷰 확인 및 인덱스 모드 선택">
    **프리뷰 청크**를 통해 실제로 문서가 어떻게 나뉘는지 확인합니다. 이후 **인덱스 모드**를 선택합니다.

    | 인덱스 모드 | 특징 |
    |---|---|
    | 고품질 | 더 정확한 검색, 처리 시간 길고 비용 높음 |
    | 경제적 | 빠른 처리, 간단한 문서에 적합 |
  </Step>
  <Step title="리랭크 및 검색 파라미터 설정">
    - **리랭크 모델 활성화**: 검색 결과를 재정렬하여 정확도 향상
    - **Top-K**: 검색할 청크의 최대 개수
    - **Top-N**: 최종적으로 AI에게 전달할 청크 수
    - **유사도 임계값**: 이 값 이상인 청크만 사용 (0~1 사이 값)
  </Step>
  <Step title="최종 확인 → 지식베이스 생성">
    설정 내용을 최종 확인한 후 **지식베이스 생성** 버튼을 클릭합니다.
  </Step>
</Steps>

## 검증 TIP

<Tip>
  지식베이스 생성 후 반드시 테스트 질문을 해보세요.

  - **테스트 프롬프트**: "문서 내용을 기반으로 답변해줘"
  - **확인 항목**: 답변 하단에 **참조 문서명과 페이지 번호**가 표시되는지 확인합니다. 출처가 표시되면 RAG가 정상 작동하고 있는 것입니다.
</Tip>

## 비교 모드

RAG의 효과를 직접 눈으로 비교할 수 있는 기능입니다.

<Steps>
  <Step title="비교 모드 탭 진입">
    **지식베이스 메뉴** → **비교 모드** 탭을 클릭합니다.
  </Step>
  <Step title="패널 A / B 설정">
    좌우 패널 각각에 비교할 방식을 선택합니다.

    | 선택지 | 설명 |
    |---|---|
    | 순수 LLM | 문서 없이 AI 자체 지식만으로 답변 |
    | 문서 참조 | 업로드 문서를 참조하여 답변 (RAG 적용) |
    | RAG 설정 | 특정 RAG 파라미터 설정값으로 답변 |
  </Step>
  <Step title="추천 비교 조합으로 실습">
    아래 조합으로 실습하면 RAG 효과를 체감할 수 있습니다.

    - **RAG 효과 체험**: 패널 A → 순수 LLM / 패널 B → 문서 참조
    - **정확도 차이 확인**: 패널 A → Top-K 3 / 패널 B → Top-K 10
  </Step>
</Steps>

## 효과적으로 활용하기

<CardGroup cols={2}>
  <Card title="KB 연결 프롬프트" icon="link">
    항상 **"첨부 문서를 참고하여 답변해주세요"** 라는 지시를 프롬프트에 포함하세요. AI가 문서를 우선 참조하도록 유도합니다.
  </Card>
  <Card title="소스 인용 요청" icon="quote-left">
    **출처와 문서 내 위치를 명시**해달라고 요청하세요. 답변의 신뢰도를 높이고, 원문 확인도 용이합니다.
  </Card>
  <Card title="문서 구조화" icon="list">
    업로드할 문서에 **목차, 제목, 단락 구분**을 명확히 해두세요. 청킹 품질이 향상되고 검색 정확도가 올라갑니다.
  </Card>
  <Card title="정기 업데이트" icon="rotate">
    문서 내용이 바뀌면 **주기적으로 문서를 갱신하고 재인덱싱**하세요. 오래된 문서로는 최신 정보를 제공할 수 없습니다.
  </Card>
</CardGroup>

---

<Note>
  RAG는 AI의 환각(Hallucination) 문제를 줄이는 가장 실용적인 방법 중 하나입니다. 신뢰할 수 있는 문서를 잘 정리하여 연결하는 것이 좋은 RAG 활용의 핵심입니다.
</Note>
