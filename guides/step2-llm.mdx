## LLM 활용이란?

**LLM 활용**은 GrowFit 플랫폼의 핵심 기능입니다. 다양한 대형 언어 모델(LLM)을 직접 사용해보고 동시에 비교할 수 있으며, **최대 3개 모델을 동시에 비교**하는 것도 가능합니다.

> **같은 질문에 모델마다 다른 답변** — 상황별 최적 모델을 선택하는 안목을 키울 수 있습니다.

---

## 왜 여러 모델을 비교해야 하나요?

<CardGroup cols={3}>
  <Card title="학습 데이터의 차이" icon="database">
    각 LLM은 서로 다른 데이터로 학습되어 **강점이 다릅니다.** 같은 질문에도 지식의 깊이나 최신성이 다를 수 있습니다.
  </Card>
  <Card title="작업 유형별 최적 모델" icon="sliders">
    코딩, 글쓰기, 요약, 논리 추론 등 **작업 유형에 따라 잘 맞는 모델이 다릅니다.** 무조건 최신 모델이 정답이 아닙니다.
  </Card>
  <Card title="최고의 AI는 없다" icon="trophy">
    **"최고의 AI"는 없으며, "이 작업에 맞는 AI"를 찾는 것이 핵심입니다.** 비교 실습을 통해 상황 판단 능력을 개발하세요.
  </Card>
</CardGroup>

## 기본 설정 vs 옵션 설정

<Tabs>
  <Tab title="기본 설정 (Basic)">
    **LLM 모델만 선택하여 바로 대화를 시작합니다.**

    - 모델 선택 후 즉시 대화 가능
    - 별도의 파라미터 조정 없이 기본값으로 동작
    - 빠른 실습과 단순 비교에 적합

    <Note>
      처음 LLM을 사용하는 경우라면 기본 설정으로 시작하여 각 모델의 특성을 먼저 파악하는 것을 권장합니다.
    </Note>
  </Tab>
  <Tab title="옵션 설정 (Advanced)">
    **스타일 프리셋으로 AI 응답을 자동 최적화합니다.**

    - **Temperature**: 응답의 창의성과 다양성 조절
    - **Top P**: 단어 선택의 다양성 범위 설정
    - **Max Length**: 최대 응답 길이 제한

    스타일 프리셋을 선택하면 이 3가지 값이 자동으로 최적화됩니다.

    <Tip>
      어떤 파라미터를 어떻게 설정해야 할지 모르겠다면 스타일 프리셋을 먼저 선택해보세요. 프리셋이 상황에 맞는 값을 자동으로 설정해줍니다.
    </Tip>
  </Tab>
</Tabs>

## 스타일 프리셋 상세

스타일 프리셋은 **정확성과 창의성** 사이의 스펙트럼 위에 위치합니다.

| 프리셋 | 특징 | 적합한 용도 |
|---|---|---|
| 정확한 (Precise) | 낮은 Temperature · 일관된 답변 | 코드 생성, 사실 확인, 요약 |
| 균형 잡힌 (Balanced) | 기본값 · 자연스러운 대화 | 일반 질의응답, 설명 요청 |
| 창의적 (Creative) | 높은 Temperature · 다양한 아이디어 | 브레인스토밍, 글쓰기, 아이디어 발굴 |

---

## 핵심 파라미터 상세 설명

AI 응답 스타일을 결정하는 3가지 핵심 값입니다. 에어컨 온도나 스피커 볼륨처럼 직관적으로 조절할 수 있습니다.

<AccordionGroup>
  <Accordion title="Temperature — 창의성 조절">
    **창의성과 무작위성의 정도를 조절합니다.**

    - **낮을수록 (0에 가깝게)**: 예측 가능하고 일관된 답변. 항상 비슷한 내용을 출력합니다.
    - **높을수록 (1~2)**: 다양하고 창의적인 답변. 같은 질문에도 매번 다른 내용이 나올 수 있습니다.

    | 값 범위 | 특성 | 추천 사용 예 |
    |---|---|---|
    | 0.0 ~ 0.3 | 매우 일관됨 | 코드 생성, 번역 |
    | 0.4 ~ 0.7 | 균형 잡힘 | 일반 대화, 설명 |
    | 0.8 ~ 1.5 | 창의적 | 소설 쓰기, 브레인스토밍 |
  </Accordion>
  <Accordion title="Top P — 단어 선택 범위">
    **다음 단어를 선택할 때 고려하는 확률 범위를 설정합니다.**

    - **낮을수록**: 확률이 높은 단어만 선택 → 안전하고 예측 가능
    - **높을수록**: 더 다양한 단어 풀에서 선택 → 풍부하고 다양한 표현

    일반적으로 Temperature와 함께 조절하며, 둘 다 높이면 창의성이 크게 올라갑니다.
  </Accordion>
  <Accordion title="Max Length — 응답 길이 제한">
    **AI가 생성하는 응답의 최대 토큰 수를 설정합니다.**

    - **낮게 설정**: 짧고 간결한 답변 (요약, 간단한 질의에 적합)
    - **높게 설정**: 길고 상세한 답변 (심층 분석, 긴 글 작성에 적합)

    <Note>
      토큰(Token)은 AI가 텍스트를 처리하는 단위입니다. 한국어는 영어보다 토큰 소비가 많으므로, 긴 답변이 필요할 때는 Max Length를 충분히 높게 설정하세요.
    </Note>
  </Accordion>
</AccordionGroup>

## 모델별 특성 비교

같은 프롬프트를 입력했을 때 각 모델이 어떻게 다르게 반응하는지 파악해두면 상황별로 최적 모델을 빠르게 선택할 수 있습니다.

<CardGroup cols={3}>
  <Card title="GPT-4o-mini" icon="bolt">
    **실행 중심의 해결사**

    - 코드 예제를 최우선으로 제시
    - 다양한 접근법 동시 제공
    - 오류 메시지 디버깅에 강점
    - 빠른 답변, 실용적 해결책 중심
  </Card>
  <Card title="Claude" icon="graduation-cap">
    **논리적인 선생님**

    - 개념 설명을 먼저 진행 후 실습
    - 구조적이고 단계적 접근
    - 시각적 묘사와 비유 활용
    - 깊이 있는 설명, 교육적 톤
  </Card>
  <Card title="Gemini" icon="layer-group">
    **핵심 요약 전문가**

    - 간결하고 핵심만 압축 제공
    - 관련 리소스 및 링크 연결
    - 최신 트렌드 및 정보 반영
    - 빠른 요약과 정보 탐색에 강점
  </Card>
</CardGroup>

## 비교 실습 시나리오

다음 프롬프트로 3개 모델을 동시에 실습해보세요.

<Tabs>
  <Tab title="코딩 학습">
    **프롬프트 예시**:
    ```
    파이썬에서 리스트와 딕셔너리의 차이점을 설명해줘.
    각각 어떤 상황에서 사용하면 좋은지 예제 코드와 함께 알려줘.
    ```

    **예상 차이점**:
    - GPT: 코드 블록 먼저, 설명은 코드 주석으로
    - Claude: 개념 차이 설명 → 표로 비교 → 예제 코드 순서
    - Gemini: 한 줄 요약 → 핵심 차이 → 간단한 예제
  </Tab>
  <Tab title="글쓰기 & 창작">
    **프롬프트 예시**:
    ```
    인공지능이 미래 직업에 미치는 영향에 대한
    설득력 있는 짧은 에세이를 써줘. (300자 이내)
    ```

    **예상 차이점**:
    - GPT: 균형 잡힌 논조, 구체적 통계 인용 시도
    - Claude: 풍부한 비유와 스토리텔링 활용
    - Gemini: 핵심 주장 압축, 간결하고 임팩트 있는 문장
  </Tab>
  <Tab title="정보 분석">
    **프롬프트 예시**:
    ```
    ChatGPT, Claude, Gemini의 주요 차이점을
    표로 비교해줘. 각 모델의 강점과 약점을 포함해줘.
    ```

    **예상 차이점**:
    - GPT: 상세한 표, 각 항목에 점수 부여 시도
    - Claude: 표 + 항목별 보충 설명 추가
    - Gemini: 깔끔한 표, 공식 출처 정보 반영
  </Tab>
</Tabs>

---

<Note>
  같은 프롬프트라도 모델마다 접근법이 전혀 다릅니다. "어떤 모델이 더 좋다"가 아니라, **"이 상황에는 어떤 모델이 더 적합한가"**를 판단하는 안목을 기르는 것이 LLM 비교 실습의 핵심 목표입니다.
</Note>
