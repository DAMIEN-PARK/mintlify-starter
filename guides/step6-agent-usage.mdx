## 워크플로우 설계 핵심 5단계

<Steps>
  <Step title="노드 추가">
    좌측 **노드 팔레트**에서 원하는 노드를 **캔버스로 드래그**하여 배치합니다.
  </Step>
  <Step title="노드 연결">
    노드의 **출력 포트**를 클릭한 채 다음 노드의 **입력 포트**로 드래그하면 화살표로 연결됩니다.
  </Step>
  <Step title="속성 설정">
    노드를 클릭하면 **우측 속성 패널**이 열립니다. 각 노드의 프롬프트, 모델, 도구 등을 이곳에서 설정합니다.
  </Step>
  <Step title="워크플로우 검증">
    **미리보기 패널**에서 실행 결과를 확인합니다.

    <Warning>
      검증에 실패한 워크플로우는 저장되지 않습니다. 오류 노드를 클릭하여 원인을 확인하고 수정한 후 재검증하세요.
    </Warning>
  </Step>
  <Step title="저장 & 게시">
    최종 확인 후 **저장**하고, 사용 범위를 선택하여 **게시**합니다.

    | 게시 옵션 | 설명 |
    |---|---|
    | 나만 사용 | 본인만 에이전트에 접근 가능 |
    | 수업에 공유 | 수강 중인 강의의 수강생과 공유 |
  </Step>
</Steps>

## 스킬(Skill) 개념

**스킬**은 에이전트 내부에 포함되는 역할 부품으로, 여러 에이전트에서 **재사용**할 수 있습니다.

<CardGroup cols={2}>
  <Card title="에이전트" icon="robot">
    **완성된 AI 도우미**

    전체 워크플로우를 포함한 하나의 완성된 서비스입니다.
  </Card>
  <Card title="스킬" icon="puzzle-piece">
    **에이전트 내 역할 부품**

    특정 기능만 담당하며, 다양한 에이전트에 재사용됩니다.
  </Card>
</CardGroup>

### skill.md 구성 요소

| 항목 | 설명 |
|---|---|
| name | 스킬 이름 |
| description | 스킬 기능 설명 (AI가 언제 이 스킬을 쓸지 판단하는 기준) |
| trigger | 스킬이 실행되는 조건 |
| model | 사용할 LLM 모델 |
| temperature | 창의성 조절 파라미터 |
| 역할 및 지시사항 | AI에게 부여할 구체적인 역할과 행동 규칙 |

### 스킬의 3가지 가치

<CardGroup cols={3}>
  <Card title="재사용성" icon="recycle">
    한 번 만든 스킬을 여러 에이전트에서 반복 활용
  </Card>
  <Card title="일관성" icon="equals">
    동일한 스킬은 항상 같은 방식으로 동작하여 결과 품질 유지
  </Card>
  <Card title="유지보수" icon="screwdriver-wrench">
    스킬 하나만 수정하면 연결된 모든 에이전트에 반영됨
  </Card>
</CardGroup>

## 스킬 만들기 (AI 도움받기)

<Steps>
  <Step title="AI 도움받기 탭 열기">
    **에이전트 빌더 → 스킬 노드 → "AI 도움받기"** 탭을 클릭합니다.
  </Step>
  <Step title="기능 설명 입력">
    만들고 싶은 스킬을 자연어로 설명합니다.

    ```
    예시: "긴 문서를 핵심만 요약해주는 도우미. 3줄 이내로 요약하고 핵심 키워드를 추출해줘."
    ```

    <Tip>
      좋은 스킬을 만들기 위한 설명 작성 팁:
      - **구체적으로**: "요약해줘" 보다는 "3줄 이내로 요약해줘"
      - **출력 형식 명시**: "번호 목록으로", "표 형식으로" 등
      - **규칙/제약 포함**: "전문 용어를 쉽게 풀어서", "한국어로만 답변"
      - **예시 제공**: 원하는 출력 예시를 함께 입력하면 정확도가 높아짐
    </Tip>
  </Step>
  <Step title="스킬 생성 클릭">
    **"스킬 생성"** 버튼을 클릭하면 AI가 skill.md를 자동으로 작성합니다.
  </Step>
  <Step title="품질 향상 후 저장">
    생성된 스킬을 검토하고 **"예시 추가"** 또는 **"개선하기"**로 품질을 높인 후 저장합니다.
  </Step>
</Steps>

## 스킬 저장 & 라이브러리

### 저장 3단계

<Steps>
  <Step title="내 스킬에 저장 클릭">
    스킬 편집 화면에서 **"내 스킬에 저장"** 버튼을 클릭합니다.
  </Step>
  <Step title="이름 & 설명 입력">
    라이브러리에서 찾기 쉬운 **이름과 설명**을 입력합니다.
  </Step>
  <Step title="저장하기">
    **"저장하기"** 버튼을 눌러 스킬 라이브러리에 등록합니다.
  </Step>
</Steps>

### 라이브러리 활용

| 기능 | 설명 |
|---|---|
| 검색 / 정렬 | 이름 또는 설명으로 스킬을 빠르게 검색 |
| 즉시 적용 | 체크박스로 선택 후 현재 에이전트에 바로 적용 |
| 수정 / 관리 | 저장된 스킬을 언제든지 수정 및 업데이트 |
| 복제 활용 | 비슷한 스킬은 복제 후 수정하여 빠르게 변형 |

### 에이전트에서 스킬 사용하기

<Steps>
  <Step title="스킬 노드 추가">캔버스에 스킬 노드를 드래그합니다.</Step>
  <Step title="내 스킬 선택">노드 속성 패널에서 저장된 스킬을 선택합니다.</Step>
  <Step title="노드 연결">이전/다음 노드와 화살표로 연결합니다.</Step>
  <Step title="실행 확인">미리보기에서 스킬이 정상 작동하는지 확인합니다.</Step>
</Steps>

## 스킬 활용 패턴

<Tabs>
  <Tab title="단일 스킬 실행">
    **하나의 스킬을 독립적으로 실행하는 가장 기본적인 패턴입니다.**

    ```
    시작 → 프롬프트 → 문서 요약 스킬 → 종료
    ```

    입력된 문서를 요약 스킬 하나로 처리하는 단순하고 명확한 구조입니다.
  </Tab>
  <Tab title="스킬 체이닝">
    **여러 스킬을 순서대로 연결하여 복합 작업을 처리합니다.**

    ```
    시작 → 번역 스킬 → 교정 스킬 → 설명 스킬 → 종료
    ```

    외국어 문서를 번역 → 교정 → 한국어 설명 순서로 자동 처리합니다.
  </Tab>
  <Tab title="서브 에이전트">
    **메인 에이전트가 필요에 따라 전문 스킬을 호출합니다.**

    ```
    메인 에이전트 → 분류 노드 → [전문가 스킬 A / 전문가 스킬 B] → 종료
    ```

    사용자 요청 유형에 따라 가장 적합한 전문 스킬로 분기합니다.
  </Tab>
</Tabs>

## 테스트 & 실행

### 워크플로우 검증

**"검증하기"** 버튼을 클릭하면 노드 순서대로 실행하며 각 단계의 상태를 표시합니다.

| 상태 | 의미 |
|---|---|
| 완료 | 노드가 정상 실행됨 |
| 검사 중 | 현재 처리 중 |
| 대기 | 이전 노드 완료 대기 중 |
| 오류 | 실행 중 문제 발생 |

### 오류 디버깅

<Steps>
  <Step title="실패 노드 클릭">오류 상태인 노드를 클릭합니다.</Step>
  <Step title="오류 내용 확인">속성 패널에서 오류 메시지와 원인을 확인합니다.</Step>
  <Step title="수정 후 재검증">문제를 수정하고 다시 검증하기를 실행합니다.</Step>
</Steps>

### 게시

검증을 통과한 워크플로우는 **"게시"** 버튼을 클릭하여 배포합니다.

## 효과적인 에이전트 구축 팁

<AccordionGroup>
  <Accordion title="1. 템플릿에서 시작하기">
    11가지 기본 템플릿 중 목적에 맞는 것을 **복제**한 뒤 수정하세요. 처음부터 만드는 것보다 훨씬 빠르고, 좋은 구조적 참고 자료가 됩니다.
  </Accordion>
  <Accordion title="2. MCP 도구와 조합하기">
    에이전트에 MCP 도구 노드를 추가하면 실시간 데이터를 활용하는 강력한 AI를 만들 수 있습니다. 웹 검색 + 요약 스킬 조합이 대표적인 예입니다.
  </Accordion>
  <Accordion title="3. 가드레일 노드 활용하기">
    수강생이나 일반 사용자에게 배포할 에이전트에는 반드시 **가드레일 노드**를 추가하세요. 부적절한 입력과 출력을 필터링하여 안전한 사용 환경을 만들 수 있습니다.
  </Accordion>
  <Accordion title="4. 단계적 복잡도 증가">
    처음에는 **시작 → 프롬프트 → 종료**의 3노드 구조로 시작하세요. 작동이 확인되면 지식베이스, MCP, 분류 노드를 하나씩 추가하며 기능을 확장하는 것이 가장 효율적입니다.
  </Accordion>
</AccordionGroup>
