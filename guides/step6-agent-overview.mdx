## 에이전트 빌더란?

**에이전트 빌더**는 코딩 없이 노드(Node)를 연결하여 AI 도구를 시각적으로 설계하는 **워크플로우 도구**입니다.

<CardGroup cols={3}>
  <Card title="노드 기반 워크플로우" icon="diagram-project">
    시작부터 종료까지 각 단계를 블록 쌓듯 연결합니다.
  </Card>
  <Card title="데이터 처리 흐름" icon="arrow-right-arrow-left">
    입력 → 프롬프트 → 스킬 / MCP / RAG → 결과 순으로 데이터가 흐릅니다.
  </Card>
  <Card title="자율 실행" icon="robot">
    AI가 판단하여 필요한 도구를 스스로 호출합니다.
  </Card>
</CardGroup>

---

## 활용 4단계

<Steps>
  <Step title="개념 이해">
    노드의 종류와 역할, 워크플로우의 흐름을 이해합니다.
  </Step>
  <Step title="설계">
    캔버스에 노드를 배치하고 연결하여 워크플로우를 설계합니다.
  </Step>
  <Step title="테스트">
    미리보기 패널에서 실행하여 결과를 확인하고 수정합니다.
  </Step>
  <Step title="배포 & 공유">
    검증이 완료된 에이전트를 게시하고 수업 또는 팀과 공유합니다.
  </Step>
</Steps>

## 기본 워크플로우 노드 6종

모든 에이전트 설계에 사용되는 핵심 노드입니다.

<AccordionGroup>
  <Accordion title="시작 노드 (Start) — 필수">
    **사용자의 초기 입력 메시지를 수신합니다.**

    - 모든 워크플로우의 시작점
    - 반드시 포함되어야 하는 필수 노드
    - 사용자 입력을 다음 노드로 전달
  </Accordion>
  <Accordion title="종료 노드 (End) — 필수">
    **최종 결과를 사용자에게 출력합니다.**

    - 모든 흐름의 끝점
    - 워크플로우의 출력 결과를 사용자에게 표시
  </Accordion>
  <Accordion title="프롬프트 노드 (Prompt)">
    **LLM에게 구체적인 지시와 역할을 부여합니다.**

    - AI의 행동 방식, 출력 형식, 제약 조건 등을 설정
    - 에이전트의 성격과 전문성을 결정하는 핵심 노드
  </Accordion>
  <Accordion title="스킬 노드 (Skill)">
    **미리 정의된 기능을 실행합니다.**

    - 퀴즈 생성, 문법 교정 등 반복 사용되는 기능을 재사용
    - 스킬 라이브러리에서 선택하거나 새로 만들어 적용
  </Accordion>
  <Accordion title="지식베이스 노드 (Knowledge)">
    **업로드된 문서에서 관련 내용을 검색합니다(RAG).**

    - 지식베이스에 등록된 파일을 연결
    - 사용자 질문과 관련된 문서 청크를 자동으로 검색하여 AI에게 전달
  </Accordion>
  <Accordion title="MCP 도구 노드 (MCP)">
    **외부 서비스와 연동하여 실시간 데이터를 활용합니다.**

    - 웹 검색, 계산, 날씨 조회 등 MCP 도구를 워크플로우에 삽입
    - AI가 도구 호출 여부를 자율적으로 판단
  </Accordion>
</AccordionGroup>

## 로직 & 고급 노드 5종

복잡한 흐름 제어와 데이터 처리에 사용하는 노드입니다.

<CardGroup cols={2}>
  <Card title="분류 노드 (Classification)" icon="sitemap">
    사용자 입력의 **의도를 분석**하여 경로를 자동으로 분기합니다. 예: "질문인가? 요청인가?"에 따라 다른 흐름으로 연결
  </Card>
  <Card title="조건분기 노드 (If/Else)" icon="code-branch">
    **변수 값이나 조건**에 따라 워크플로우 흐름을 제어합니다. 프로그래밍의 if-else 구문과 동일한 역할
  </Card>
  <Card title="변환 노드 (Transform)" icon="shuffle">
    **데이터 포맷을 변경**하거나 특정 값을 추출합니다. JSON 파싱, 텍스트 정제, 필드 추출 등에 활용
  </Card>
  <Card title="상태설정 노드 (Set State)" icon="database">
    **대화 문맥과 사용자 정보를 변수에 저장**합니다. 이전 단계의 결과를 이후 단계에서 활용 가능
  </Card>
  <Card title="가드레일 노드 (Guardrail)" icon="shield-halved">
    **입출력을 검사하여 부적절한 내용을 필터링**합니다. 안전하고 신뢰할 수 있는 에이전트 운영에 필수
  </Card>
</CardGroup>

## 핵심 개념 4가지

<AccordionGroup>
  <Accordion title="워크플로우 (Workflow)">
    노드들이 연결된 전체 흐름입니다. 하나의 에이전트는 하나의 워크플로우로 구성됩니다. 시작 노드에서 입력을 받아 종료 노드에서 결과를 출력하기까지의 전체 경로를 의미합니다.
  </Accordion>
  <Accordion title="노드 (Node)">
    워크플로우를 구성하는 개별 처리 단위입니다. 각 노드는 특정 역할(LLM 호출, 도구 실행, 조건 판단 등)을 수행하고 결과를 다음 노드로 전달합니다.
  </Accordion>
  <Accordion title="스킬 (Skill)">
    에이전트 내부에서 재사용할 수 있는 역할 부품입니다. 한 번 만들어 두면 여러 에이전트에서 불러와 사용할 수 있어 일관성과 유지보수 효율이 높아집니다.
  </Accordion>
  <Accordion title="세션 (Session)">
    사용자와 에이전트 사이의 단일 대화 단위입니다. 세션이 유지되는 동안 상태설정 노드에 저장된 변수가 유효하며, 대화 문맥이 이어집니다.
  </Accordion>
</AccordionGroup>

## MCP 도구 vs 에이전트 비교

| 구분 | MCP 도구 | 에이전트 |
|---|---|---|
| 비유 | AI에게 쥐여주는 도구 (연필, 계산기) | AI의 행동 흐름 설계 (업무 매뉴얼, 순서도) |
| 역할 | 특정 기능 실행 | 전체 작업 흐름 조율 |
| 관계 | 에이전트 안에서 MCP 노드로 사용됨 | MCP를 포함한 더 큰 실행 단위 |

<Tip>
  MCP와 에이전트는 대립하는 개념이 아닙니다. 에이전트 워크플로우 안에 MCP 도구 노드를 삽입하여 함께 활용하는 것이 가장 강력한 조합입니다.
</Tip>

<Note>
  에이전트 빌더는 복잡한 AI 업무 자동화를 코딩 없이 구현할 수 있는 강력한 도구입니다. 처음에는 기본 노드 6종만으로 시작하고, 익숙해지면 로직 노드와 MCP를 조합해보세요.
