## 프롬프트 엔지니어링이란?

**프롬프트 엔지니어링**은 AI에게 효과적으로 지시하는 기술입니다. 잘 작성된 프롬프트 하나만으로도, 같은 모델에서 훨씬 더 좋은 결과를 얻을 수 있습니다.

> **"같은 모델도 프롬프트에 따라 전혀 다른 결과를 냅니다."**

---

## 핵심 개념

<CardGroup cols={2}>
  <Card title="프롬프트 (Prompt)" icon="message">
    AI에게 전달하는 **질문이나 지시문**입니다. 단순한 질문부터 복잡한 역할 부여까지 모든 AI 입력이 프롬프트에 해당합니다.
  </Card>
  <Card title="프롬프트 엔지니어링 (Prompt Engineering)" icon="wrench">
    원하는 결과를 얻기 위해 프롬프트를 **전략적으로 설계하고 최적화**하는 기술입니다.
  </Card>
</CardGroup>

---

## 프롬프트 엔지니어링의 효과

<CardGroup cols={3}>
  <Card title="정확성 향상" icon="bullseye">
    모호한 질문 대신 명확한 지시문을 작성하면 원하는 답변을 바로 얻을 수 있습니다.
  </Card>
  <Card title="일관성 유지" icon="repeat">
    동일한 프롬프트 구조를 반복 사용하면 매번 비슷한 품질의 결과를 얻습니다.
  </Card>
  <Card title="재사용성 확보" icon="recycle">
    잘 만든 프롬프트는 템플릿으로 저장하여 다양한 상황에 재활용할 수 있습니다.
  </Card>
</CardGroup>

## 기본 vs 옵션 설정

<Tabs>
  <Tab title="기본 (Basic) — 일회성 질문">
    일회성으로 AI에게 질문하거나 간단한 작업을 요청할 때 사용합니다.

    | 기법 | 설명 |
    |---|---|
    | 명확한 지시문 | 구체적으로 무엇을 원하는지 명시 |
    | 간단한 역할 부여 | AI에게 전문가 역할을 부여 |
    | 출력 형식 지정 | 원하는 결과의 형태를 지정 |

    **특징**: 입력창에 직접 작성하며, 해당 대화에만 적용됩니다.
  </Tab>
  <Tab title="옵션 (Advanced) — 반복 업무">
    반복적인 작업이나 일관된 품질이 필요한 경우에 사용합니다.

    | 기법 | 설명 |
    |---|---|
    | 시스템 프롬프트 | 대화 전체에 적용되는 AI 기본 설정 |
    | Few-shot 기법 | 예시를 제공하여 패턴 학습 |
    | 파라미터 조절 | Temperature, Top P 등 세부 조정 |
    | 템플릿 활용 | 사전 제작된 프롬프트 템플릿 사용 |

    **특징**: 시스템 프롬프트 설정창에 작성하며, 모든 대화에 지속 적용됩니다.

    <Note>
      고급 기법(시스템 프롬프트, Few-shot)은 **Step 3 고급 페이지**에서 자세히 다룹니다.
    </Note>
  </Tab>
</Tabs>

## 기본 프롬프트 작성 기법 3가지

### 기법 1: 명확한 지시문

모호한 표현 대신 **구체적이고 명확한 지시**를 작성합니다.

<Tabs>
  <Tab title="나쁜 예시 (모호함)">
    ```
    이거 설명해줘.
    ```
    ```
    파이썬 좀 알려줘.
    ```
    ```
    마케팅 전략 뭐가 있어?
    ```
  </Tab>
  <Tab title="좋은 예시 (명확함)">
    ```
    파이썬의 리스트(List)와 튜플(Tuple)의 차이점을
    코딩을 처음 배우는 초보자도 이해할 수 있게
    3문장으로 설명해줘.
    ```
    ```
    파이썬 for 반복문의 기본 문법을 알려주고,
    실생활 예시 2개를 코드로 보여줘.
    ```
    ```
    스타트업의 초기 소셜미디어 마케팅 전략을
    예산이 없는 상황을 가정하여 5가지로 정리해줘.
    ```
  </Tab>
</Tabs>

**명확한 지시문 작성 공식**:

| 구성 요소 | 설명 | 예시 |
|---|---|---|
| 대상/주제 | 무엇에 대한 설명인지 | "파이썬의 리스트" |
| 대상 독자 | 누구를 위한 설명인지 | "코딩 초보자도" |
| 형식/분량 | 어떤 형태로, 얼마나 | "3문장으로" |

---

### 기법 2: 역할 부여

AI에게 **특정 전문가의 역할**을 부여하면 해당 분야의 전문성이 반영된 답변을 얻을 수 있습니다.

<Tabs>
  <Tab title="역할 부여 예시">
    | 역할 | 프롬프트 예시 | 기대 효과 |
    |---|---|---|
    | 교육 전문가 | "당신은 10년 경력의 초등학교 선생님입니다." | 쉽고 친근한 설명 |
    | 마케팅 전문가 | "당신은 B2B SaaS 마케팅 전문가입니다." | 전문적인 마케팅 관점 |
    | 데이터 분석가 | "당신은 Python과 SQL을 전문으로 하는 데이터 분석가입니다." | 데이터 중심 접근법 |
    | 비즈니스 컨설턴트 | "당신은 McKinsey 출신의 전략 컨설턴트입니다." | 구조적 사고와 프레임워크 활용 |
    | 고객 지원 전문가 | "당신은 친절하고 공감 능력이 뛰어난 고객 서비스 담당자입니다." | 따뜻하고 해결 중심적 응대 |
  </Tab>
  <Tab title="역할 부여 전/후 비교">
    **역할 없이:**
    ```
    머신러닝 설명해줘.
    ```
    → 일반적이고 광범위한 설명

    **역할 부여 후:**
    ```
    당신은 머신러닝을 한 번도 접해보지 않은 중학생에게
    쉽게 가르치는 선생님입니다.
    머신러닝이 무엇인지 일상생활 예시를 들어 설명해줘.
    ```
    → 넷플릭스 추천, 스팸 메일 필터 등 친숙한 예시로 설명
  </Tab>
</Tabs>

---

### 기법 3: 출력 형식 지정

원하는 **결과물의 형태**를 미리 지정하면 가공 없이 바로 활용할 수 있는 결과를 얻습니다.

<AccordionGroup>
  <Accordion title="표 형식">
    ```
    GPT-4, Claude, Gemini의 특징을
    강점 / 약점 / 추천 용도 열로 구분하여 표로 정리해줘.
    ```
  </Accordion>
  <Accordion title="번호 목록 (순서 있음)">
    ```
    블로그 포스트 작성 순서를 1~7단계로
    각 단계별 핵심 작업과 함께 번호 목록으로 정리해줘.
    ```
  </Accordion>
  <Accordion title="불릿 목록 (순서 없음)">
    ```
    파이썬을 배울 때 알아야 할 핵심 라이브러리를
    각각 한 줄 설명과 함께 불릿 목록으로 나열해줘.
    ```
  </Accordion>
  <Accordion title="JSON 형식">
    ```
    다음 텍스트에서 이름, 이메일, 전화번호를 추출하여
    JSON 형식으로 반환해줘.

    [텍스트 입력]
    ```
  </Accordion>
  <Accordion title="마크다운 형식">
    ```
    React Hooks에 대한 튜토리얼을 마크다운 형식으로 작성해줘.
    제목(##), 소제목(###), 코드 블록(```)을 포함해줘.
    ```
  </Accordion>
</AccordionGroup>

## 3가지 기법 조합하기

3가지 기법을 동시에 활용하면 훨씬 강력한 결과를 얻을 수 있습니다.

<Tabs>
  <Tab title="조합 예시 1 — 교육 콘텐츠">
    ```
    당신은 마케팅 분야 10년 경력의 전문가입니다. (역할)

    소셜미디어 마케팅의 핵심 개념을
    마케팅을 처음 공부하는 대학생도 이해할 수 있게 설명해줘. (지시문)

    설명은 표 형식으로 정리하되,
    개념명 / 한 줄 정의 / 실제 사례 3개 열로 구성해줘. (형식)
    ```
  </Tab>
  <Tab title="조합 예시 2 — 코드 리뷰">
    ```
    당신은 클린 코드를 중시하는 시니어 개발자입니다. (역할)

    아래 파이썬 코드의 문제점을 찾아서
    각 문제의 원인과 개선 방법을 설명해줘. (지시문)

    결과는 문제점 / 원인 / 개선 코드 열로 구성된 표로 보여줘. (형식)

    [코드 입력]
    ```
  </Tab>
  <Tab title="조합 예시 3 — 비즈니스 분석">
    ```
    당신은 스타트업 투자 심사역입니다. (역할)

    다음 비즈니스 아이디어의 강점, 약점, 기회, 위협을
    투자자 관점에서 비판적으로 분석해줘. (지시문)

    SWOT 분석 표 형식으로 정리하고,
    마지막에 투자 가능성에 대한 한 줄 의견을 추가해줘. (형식)

    [비즈니스 아이디어 입력]
    ```
  </Tab>
</Tabs>

<Tip>
  **3가지 기법 조합 공식**: `역할 부여 + 명확한 지시문 + 출력 형식 지정`을 하나의 프롬프트에 담으면, 전문가 수준의 결과물을 일관되게 얻을 수 있습니다. 처음에는 어색하더라도 반복하다 보면 자연스럽게 몸에 익힙니다.
</Tip>

  기본 기법에 익숙해졌다면, **시스템 프롬프트와 Few-shot** 기법을 다루는 [Step 3 고급 페이지](/guides/step3-prompt-advanced)로 넘어가세요. 반복 업무와 고품질 결과물이 필요한 상황에서 활약하는 고급 기술을 배울 수 있습니다.
