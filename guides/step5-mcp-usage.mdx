## MCP 실행 프로세스

MCP가 어떤 흐름으로 작동하는지 이해하면 도구를 더 효과적으로 활용할 수 있습니다.

<Steps>
  <Step title="prompt_sent — 메시지 전달">
    사용자가 입력한 메시지와 등록된 **도구 목록**이 함께 AI에게 전달됩니다.
  </Step>
  <Step title="tool_decision — 도구 판단">
    AI가 질문을 분석하여 **필요한 도구를 자율적으로 판단**합니다. 사용자의 개입 없이 AI가 스스로 결정합니다.
  </Step>
  <Step title="tool_execution — 도구 실행">
    선택된 도구가 **외부 서버에서 실행**됩니다. (예: 날씨 API 호출, 웹 검색 실행 등)
  </Step>
  <Step title="response_generated — 답변 생성">
    도구 실행 결과를 받아 AI가 **자연어 답변으로 가공**하여 사용자에게 전달합니다.
  </Step>
</Steps>

## MCP 만들기 (외부 도구 조합)

플랫폼에서 제공하는 도구들을 조합하여 나만의 MCP를 만드는 방법입니다.

<Steps>
  <Step title="목적 설정">
    어떤 **작업 용도**로 MCP를 만들지 먼저 결정합니다. 예: 리서치 보조, 수학 계산 전용, 글쓰기 지원 등
  </Step>
  <Step title="도구 선택">
    목적에 맞는 **플랫폼 도구**를 선택합니다. 16가지 도구 중 필요한 것을 조합하세요.
  </Step>
  <Step title="이름 & 저장">
    MCP에 **이름과 설명**을 작성한 후 저장합니다. 나중에 목록에서 쉽게 찾을 수 있는 명확한 이름을 사용하세요.
  </Step>
  <Step title="조합 테스트">
    저장된 MCP를 불러와 **모델을 선택**하고, 프롬프트를 작성한 뒤 실행하여 결과를 확인합니다.
  </Step>
</Steps>

<Tip>
  검증 방법: **MCP를 OFF로 변경**한 후 같은 질문을 다시 던져보세요. 답변의 차이를 직접 비교하면 MCP 작동 여부를 확실히 확인할 수 있습니다.
</Tip>

## 커스텀 도구 만들기

플랫폼 기본 도구 외에 **직접 도구를 코딩**하여 생성할 수 있습니다.

<Note>
  코딩 경험이 없어도 괜찮습니다. **AI 도움받기** 기능이 Input Schema와 Handler Code를 자동으로 생성해줍니다.
</Note>

### 커스텀 도구 구성 요소

<Tabs>
  <Tab title="Input Schema">
    **도구의 입력값을 정의합니다.**

    - 도구가 받을 파라미터의 이름, 타입, 설명을 지정
    - **비주얼 모드**: UI로 항목을 직접 추가 (코딩 불필요)
    - **JSON 모드**: JSON 형식으로 직접 작성 (고급 사용자)
  </Tab>
  <Tab title="Handler Code">
    **도구의 실행 로직을 JavaScript로 작성합니다.**

    - 입력값을 받아 처리하는 실제 코드
    - **AI 도움받기**로 자동 생성 가능
    - 외부 API 호출, 데이터 변환 등 다양한 로직 구현 가능
  </Tab>
</Tabs>

### 커스텀 도구 생성 절차

<Steps>
  <Step title="도구 만들기 팝업 열기">
    MCP 설정 화면에서 **도구 만들기** 버튼을 클릭하여 팝업을 엽니다.
  </Step>
  <Step title="이름 & 설명 입력">
    도구의 **이름과 설명**을 입력합니다. 설명은 AI가 도구를 언제 사용할지 판단하는 데 활용되므로 구체적으로 작성하세요.
  </Step>
  <Step title="AI 도움받기로 자동 생성">
    **AI 도움받기** 버튼을 클릭하고 원하는 기능을 자연어로 설명합니다. AI가 Input Schema와 Handler Code를 자동으로 작성해줍니다.
  </Step>
  <Step title="Schema / Handler 확인">
    자동 생성된 내용을 검토합니다. 필요하면 수정하거나 AI에게 재생성을 요청하세요.
  </Step>
  <Step title="저장">
    내용이 확인되면 **저장** 버튼을 클릭하여 커스텀 도구를 등록합니다.
  </Step>
</Steps>

## 활용 팁

<AccordionGroup>
  <Accordion title="1. ON/OFF 비교 실습">
    같은 질문을 MCP ON 상태와 OFF 상태에서 각각 실행해보세요. 답변의 정확도와 최신성 차이를 직접 체감할 수 있습니다. 학습자에게 MCP의 가치를 보여주는 가장 효과적인 방법입니다.
  </Accordion>
  <Accordion title="2. 나만의 도구 만들기">
    플랫폼 기본 도구에 없는 기능이 필요하다면 커스텀 도구를 만들어보세요. 예: 특정 사이트의 데이터를 파싱하는 도구, 내부 규칙에 따른 텍스트 변환 도구 등. AI 도움받기 기능으로 코딩 없이도 만들 수 있습니다.
  </Accordion>
  <Accordion title="3. 공유 및 가져오기 (준비 중)">
    곧 출시될 공유 기능을 통해 잘 만든 MCP를 동료나 수강생과 공유할 수 있습니다. 커뮤니티에서 다른 사람이 만든 도구를 가져와 활용하는 것도 가능해질 예정입니다.
  </Accordion>
  <Accordion title="4. 에이전트 빌더 연동">
    MCP는 에이전트 빌더의 MCP 도구 노드로 연동하면 더욱 강력해집니다. Step 6 에이전트 빌더에서 만든 MCP를 워크플로우 내에 직접 삽입할 수 있습니다.
  </Accordion>
</AccordionGroup>

---

<Note>
  MCP는 단순한 도구 추가가 아니라 AI의 능력 범위를 실시간 세계로 확장하는 핵심 기술입니다. 자신의 업무에 맞는 도구 조합을 찾아 나만의 MCP 세트를 구성해보세요.
</Note>
