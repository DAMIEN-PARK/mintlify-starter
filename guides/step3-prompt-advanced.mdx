## 시스템 프롬프트 (System Prompt)

### 개념

**시스템 프롬프트**는 대화 전체에 적용되는 AI의 기본 설정입니다. 역할, 전문성, 톤, 제약 조건을 미리 정의해두면 매번 같은 내용을 반복하지 않아도 됩니다.

<Note>
  **비유로 이해하기**: 배우에게 대본을 주기 전에 "당신은 친절하고 인내심 있는 선생님 역할입니다. 항상 쉬운 말로, 예시를 들어 설명해주세요."라고 사전에 알려주는 것과 같습니다. 이 설정이 대화 내내 유지됩니다.
</Note>

### 시스템 프롬프트 4대 구성 요소

<Steps>
  <Step title="역할 (Role)">
    AI가 맡을 전문가 역할을 구체적으로 정의합니다.

    ```
    당신은 10년 경력의 시니어 소프트웨어 개발자이자
    AI 교육 전문가입니다.
    ```

    역할이 구체적일수록 답변의 전문성이 높아집니다.
  </Step>
  <Step title="전문성 (Expertise)">
    어떤 분야와 기술에 특화되어 있는지 명시합니다.

    ```
    Python, 알고리즘 설계, 코드 리뷰 및 디버깅,
    머신러닝 기초 교육을 전문으로 합니다.
    ```
  </Step>
  <Step title="톤 & 스타일 (Tone & Style)">
    어떤 방식으로 소통할지 설정합니다.

    ```
    - 친근하고 격려하는 톤 유지
    - 초보자도 이해할 수 있는 쉬운 언어 사용
    - 칭찬으로 시작하여 개선점 제안하는 방식
    - 실망시키지 않고 성취감을 느끼게 하는 피드백
    ```
  </Step>
  <Step title="제약 조건 (Constraints)">
    하지 말아야 할 것, 항상 지켜야 할 것을 정의합니다.

    ```
    - 전문 용어 사용 시 반드시 괄호 안에 설명 추가
    - 비유와 실생활 예시 필수 포함
    - 답변 길이는 500자 이내로 유지
    - 모르는 것은 모른다고 명확히 말할 것
    ```
  </Step>
</Steps>

### 완성된 시스템 프롬프트 예시

<Tabs>
  <Tab title="코딩 튜터">
    ```
    당신은 10년 경력의 시니어 개발자이자 교육 전문가입니다.

    [전문 분야]
    Python, 알고리즘, 자료구조, 코드 디버깅

    [소통 방식]
    - 친근하고 격려하는 톤 유지
    - 개념 먼저 설명 후 예제 코드 제시
    - 초보자도 이해할 수 있게 비유 활용

    [제약 사항]
    - 전문 용어 사용 시 괄호 안에 설명 필수
    - 코드 예제는 반드시 주석 포함
    - 한 번에 한 가지 개념만 집중 설명
    ```
  </Tab>
  <Tab title="고객 서비스 에이전트">
    ```
    당신은 IT 서비스 회사의 친절한 고객 지원 담당자입니다.

    [역할]
    고객 문의 접수, 문제 해결 안내, 에스컬레이션 판단

    [소통 방식]
    - 항상 공감 표현으로 시작 ("불편을 드려 죄송합니다")
    - 고객 이름을 사용하여 개인화된 응대
    - 복잡한 기술 용어 사용 금지

    [제약 사항]
    - 환불/보상 약속은 절대 하지 말 것
    - 해결 불가 시 담당자 연결 안내
    - 모든 답변은 정중한 존댓말 사용
    ```
  </Tab>
  <Tab title="콘텐츠 작성 도우미">
    ```
    당신은 디지털 마케팅을 전문으로 하는
    콘텐츠 전략가 및 카피라이터입니다.

    [전문 분야]
    블로그 포스팅, SNS 콘텐츠, 이메일 뉴스레터, 광고 카피

    [작성 원칙]
    - 독자의 공감을 이끌어내는 스토리텔링 방식
    - SEO를 고려한 핵심 키워드 자연스럽게 포함
    - 명확한 CTA(행동 유도 문구) 포함

    [금지 사항]
    - 과장된 표현이나 근거 없는 주장 금지
    - 문장은 3줄을 넘기지 않도록
    - 수동태 대신 능동태 문장 사용
    ```
  </Tab>
</Tabs>

## Few-shot 기법

### 개념

**Few-shot** 기법은 "이런 식으로 대답해줘"라는 패턴을 **예시로 보여주어** AI가 답변 방식을 학습하게 만드는 기법입니다.

규칙을 설명하는 것보다 예시를 보여주는 것이 더 효과적일 때 사용합니다.

### Zero-shot vs One-shot vs Few-shot

| 방식 | 설명 | 특징 |
|---|---|---|
| Zero-shot | 예시 없이 지시만 | 빠르지만 일관성 낮음 |
| One-shot | 예시 1개 제공 | 어느 정도 패턴 학습 |
| Few-shot | 예시 2~5개 제공 | **가장 안정적, 권장** |

<Note>
  예시가 많을수록 정확도는 높아지지만, 그만큼 **토큰 사용량이 증가**합니다. 일반적으로 3~5개의 예시가 정확도와 효율성의 균형을 잘 맞춥니다.
</Note>

### Few-shot 실습 예시

AI에게 원하는 **입력-출력 패턴**을 예시로 보여주면, 새로운 입력에도 같은 형식으로 답변합니다.

<Tabs>
  <Tab title="감정 분석">
    ```
    다음 문장의 감정을 분석해줘.
    [긍정], [부정], [중립] 중 하나로만 답해줘.

    예시 1)
    입력: "오늘 정말 최고의 하루였어요!"
    출력: [긍정]

    예시 2)
    입력: "배송이 3일이나 늦어서 너무 실망했어요."
    출력: [부정]

    예시 3)
    입력: "오늘 날씨는 맑습니다."
    출력: [중립]

    이제 분석해줘:
    입력: "서비스는 괜찮은데 가격이 좀 비싸네요."
    출력:
    ```
  </Tab>
  <Tab title="데이터 변환">
    ```
    자연어 문장을 JSON 형식으로 변환해줘.

    예시 1)
    입력: "김철수, 30세, 서울 거주, 개발자"
    출력: {"name": "김철수", "age": 30, "city": "서울", "job": "개발자"}

    예시 2)
    입력: "이영희, 25세, 부산 거주, 디자이너"
    출력: {"name": "이영희", "age": 25, "city": "부산", "job": "디자이너"}

    이제 변환해줘:
    입력: "박민준, 35세, 인천 거주, 마케터"
    출력:
    ```
  </Tab>
  <Tab title="요약 스타일 지정">
    ```
    아래 기사를 "핵심 한 줄 + 주요 포인트 3개" 형식으로 요약해줘.

    예시 1)
    원문: [AI 기술 발전 관련 긴 기사...]
    출력:
    핵심: AI 기술이 2024년 대비 3배 빠르게 발전 중
    • GPT-5 출시로 추론 능력 대폭 향상
    • 기업 AI 도입률 67%로 전년 대비 2배 증가
    • 규제 프레임워크 정비 시급성 부각

    이제 요약해줘:
    원문: [요약할 기사 입력]
    출력:
    ```
  </Tab>
</Tabs>

### 직접 Few-shot 프롬프트 만들기

<Steps>
  <Step title="원하는 출력 형식 결정">
    먼저 AI에게 원하는 **최종 결과물의 형태**를 명확히 정합니다.
    예: 표, JSON, 특정 문장 구조, 이모지 포함 여부 등
  </Step>
  <Step title="입력-출력 쌍 2~5개 작성">
    직접 2~5개의 **입력과 기대 출력 예시**를 작성합니다.
    예시가 다양할수록 AI가 패턴을 더 잘 학습합니다.
  </Step>
  <Step title="프롬프트에 예시 삽입">
    "예시 1) / 예시 2) / ..." 형식으로 예시를 프롬프트 앞부분에 배치합니다.
  </Step>
  <Step title="실제 입력 추가">
    예시들 다음에 실제로 처리할 입력을 추가하고 `출력:` 또는 `결과:`를 입력하면 AI가 패턴에 맞게 답변합니다.
  </Step>
</Steps>

## 템플릿 불러오기

GrowFit 플랫폼에는 사전 제작된 Few-shot 템플릿이 준비되어 있습니다.

<Steps>
  <Step title="입력창 [+] 버튼 클릭">
    AI 실습 화면 하단 입력창 왼쪽의 **[+] 버튼**을 클릭합니다.
  </Step>
  <Step title="'파일 & 문서' 선택">
    드롭다운 메뉴에서 **파일 & 문서** 항목을 선택합니다.
  </Step>
  <Step title="'템플릿 사용' 선택">
    파일 첨부 창에서 **템플릿 사용** 탭으로 이동합니다.
  </Step>
  <Step title="Few-shot 탭 → 예시 선택">
    **Few-shot 탭**을 클릭하면 다양한 카테고리별 예시 템플릿 목록이 표시됩니다. 원하는 템플릿을 선택하면 입력창에 자동으로 불러와집니다.
  </Step>
</Steps>

<Tip>
  템플릿을 불러온 후 입력-출력 예시 부분을 자신의 업무나 학습 내용에 맞게 수정하여 **커스텀 Few-shot 프롬프트**를 만들어보세요. 한번 잘 만든 프롬프트는 반복 사용하면 업무 효율이 크게 높아집니다.
</Tip>

## 시스템 프롬프트 + Few-shot 조합하기

두 기법을 함께 사용하면 최고 수준의 일관성과 품질을 달성할 수 있습니다.

```
[시스템 프롬프트]
당신은 고객 리뷰를 분석하는 데이터 분석 전문가입니다.
항상 JSON 형식으로만 답변하세요.
감정은 positive / negative / neutral 중 하나만 사용하세요.

[Few-shot 예시]
리뷰: "배송이 빠르고 포장도 꼼꼼해서 만족합니다."
분석: {"sentiment": "positive", "key_point": "빠른 배송, 꼼꼼한 포장", "score": 5}

리뷰: "품질은 괜찮은데 가격이 너무 비싸요."
분석: {"sentiment": "neutral", "key_point": "적정 품질, 높은 가격", "score": 3}

[실제 입력]
리뷰: "제품 설명과 달리 색상이 완전히 다르게 왔어요. 교환 요청했습니다."
분석:
```

<Warning>
  시스템 프롬프트와 Few-shot을 함께 사용할 때는 두 설정이 **서로 충돌하지 않도록** 주의하세요. 시스템 프롬프트에서 정의한 제약 조건이 Few-shot 예시와 일관성을 유지해야 합니다.
</Warning>

---

<Note>
  프롬프트 엔지니어링은 한 번에 완성되지 않습니다. **실험 → 결과 확인 → 개선**의 반복 과정을 통해 점점 더 좋은 프롬프트를 만들어갑니다. GrowFit의 비교 모드를 활용하여 프롬프트 변경 전/후 결과를 직접 비교해보세요.
</Note>
