강사 모드 주요 역할
강의 개설 및 운영
강의 기간 설정, 참여 학생 수 제한, 수업에 사용할 AI 모델(GPT-4o, Claude 3.5 등)을 직접 선택하고 관리합니다.
학생 초대 및 관리
초대 코드 및 링크를 발급하여 학생을 등록하고, 개별 학생의 학습 진도와 이력을 실시간으로 조회합니다.
학습 자료 제공
수업에 필요한 맞춤형 에이전트, 프롬프트 템플릿, 
Few-shot 예시 세트를 구성하여 학생들에게 공유합니다.
MCP 구성 및 배포
외부 도구(검색, 계산기 등) 조합을 생성하고 학생 실습 환경에 배포합니다.
에이전트 설계
노드 기반 워크플로우를 직접 설계하고 커스텀 에이전트를 클래스에 배포합니다.
사용량 모니터링
AI 모델별 토큰 사용량과 학생별 활동 리포트를 시각화된 차트로 확인합니다.

에이전트 핵심 개념 4가지
에이전트 빌더의 핵심 개념을 이해하면 워크플로우 설계가 쉬워집니다.
1
워크플로우
노드(블록)와 연결선으로 이루어진 작업 흐름입니다. 시작부터 끝까지 AI가 어떤 순서로 일할지 정하는 설계도입니다.
2
노드
워크플로우의 각 단계를 담당하는 블록입니다. 프롬프트, 분류, 도구 호출 등 다양한 종류가 있습니다.
3
스킬
재사용 가능한 AI 능력 모듈입니다. 
한 번 만들면 여러 에이전트에서 반복해서 사용할 수 있습니다.
4
세션
에이전트와의 연속 대화 단위입니다. 이전 대화 내용을 기억하며 맥락을 이어갑니다.
MCP 도구 vs 에이전트 비교
구분
MCP 도구 (Step 5)
에이전트 빌더 (Step 6)
역할
AI에게 쥐여주는 도구
AI의 행동 흐름을 설계
비유
연필, 계산기, 사전 같은 도구
업무 매뉴얼, 순서도 같은 절차
관계
에이전트 안에서 MCP 노드로 사용
MCP 도구를 워크플로우의 한 단계로 활용
